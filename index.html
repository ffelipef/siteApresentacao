PROBLEMA E MOTIVAÇÃO: MODELANDO O MUNDO REAL

Nosso desafio foi encontrar o caminho mais curto entre quaisquer dois pontos em Belém do Pará. Essa é uma necessidade fundamental para sistemas de navegação por GPS, otimização de rotas para entregas e planejamento logístico.

O que é um grafo?

Um grafo é uma estrutura matemática composta por vértices (ou nós) e arestas (ou ligações) que conectam esses vértices. Em termos simples, um grafo pode ser visualizado como um conjunto de pontos (vértices) interligados por linhas (arestas).

Modelagem do Mapa Urbano como Grafo

  • Vértices (Nós): Cada cruzamento ou ponto de interesse no mapa de Belém foi representado como um vértice.
  • Arestas (Vias): As ruas foram modeladas como arestas, conectando os respectivos cruzamentos.
  • Pesos (Distâncias): O "custo" de percorrer cada aresta foi definido pela distância em metros da rua, utilizando dados precisos do OSMnx.

CONCEITOS DE GRAFOS NA PRÁTICA: NAVEGANDO EM BELÉM

Para modelar as complexidades do trânsito urbano de Belém, aplicamos conceitos essenciais da Teoria dos Grafos:

Dígrafos (Grafo Direcionado)

Nosso mapa é um dígrafo, pois ruas de mão única significam que a direção da viagem importa. Uma aresta de A para B não implica uma aresta de B para A, refletindo a realidade do tráfego urbano.

Conectividade

Para que um caminho exista entre dois pontos, eles devem estar conectados. Um fechamento de rua, por exemplo, altera a conectividade do grafo, podendo isolar partes do mapa e exigir novas rotas.

Caminho e Circuito

Um caminho é uma sequência de ruas do ponto A ao ponto B. Um circuito seria uma rota que começa e termina no mesmo local, útil para patrulhas ou entregas circulares. Entender essas estruturas é crucial para otimizar rotas.

A ESCOLHA DO ALGORITMO DE FLOYD-WARSHALL

O algoritmo de Floyd-Warshall foi o mais adequado para realizar o mapeamento urbano e calcular os menores caminhos entre dois pontos dentro da área da Alcindo Cacela.

Objetivo do Floyd-Warshall

Este algoritmo calcula o caminho mais curto entre todos os pares de vértices em um grafo. Ou seja, ele encontra a melhor rota em distância de qualquer ponto de partida para qualquer ponto de chegada.

Por que não Dijkstra?

Dijkstra é excelente para encontrar o caminho mais curto a partir de uma única origem para todos os outros destinos. Perfeito para um aplicativo de GPS que calcula a rota do seu local atual.

A Vantagem do Floyd-Warshall

Floyd-Warshall brilha em cenários onde precisamos pré-calcular todas as rotas possíveis. Para um painel de controle logístico ou um sistema que requer consulta rápida de rotas entre inúmeros pontos, o custo inicial de pré-cálculo compensa, oferecendo flexibilidade e agilidade.

ESTRUTURA DE DADOS E FERRAMENTAS

Lista de Adjacência

A escolha da Lista de Adjacência foi estratégica para o Floyd-Warshall. Este algoritmo opera eficientemente sobre uma estrutura, iterando através dela para determinar apenas as distâncias mínimas mais importantes. Dessa forma o algoritmo consegue ficar mais leve.

Ferramentas Utilizadas

  • OSMnx: Uma poderosa biblioteca Python que nos permitiu extrair dados geográficos detalhados (ruas, cruzamentos) do OpenStreetMap e convertê-los em grafos.
  • NetworkX: Essencial para a manipulação e análise do grafo obtido. O OSMnx se integra perfeitamente ao NetworkX.
  • NumPy: Fundamental para a criação e manipulação eficiente da matriz de distâncias, garantindo performance no cálculo do algoritmo.
  • Flask & Scikit-learn: Utilizados para construir a aplicação web interativa e processar os dados.

Limitações e Futuro

  • Complexidade Temporal: A complexidade de O(V^3) limita a escalabilidade para cidades inteiras.
  • Modelo Estático: O modelo não considera dados dinâmicos como tráfego em tempo real.
  • Custo de Memória: A matriz de adjacência consome muita memória para grafos grandes.

Futuras melhorias podem incluir algoritmos mais eficientes, integração de dados em tempo real e otimizações de memória.
